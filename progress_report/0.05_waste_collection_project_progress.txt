Waste Collection Route Tracker – Progress Summary

Project Title: Waste Collection Route Tracker

Core System: Smart Waste Collection Route Manager

Overview This project simulates a municipal waste collection system
using Java and Object-Oriented Programming. The system models areas,
waste bins, trucks, routes, and scheduling logic to simulate real-world
waste collection operations.

  -------------------------
  1. PROJECT ARCHITECTURE
  -------------------------

Root Package: wastecollection

Packages created:

wastecollection ├ model ├ service ├ utils └ main

Purpose of each package:

model Contains core system entities (data models).

service Contains manager classes responsible for system logic.

utils Reserved for helper utilities.

main Contains the Main class which runs the system simulation.

  -------------------------------
  2. MODEL LAYER IMPLEMENTATION
  -------------------------------

The model package contains all core entities of the system.

Classes created:

Area Represents a geographic area where waste bins are located.
Fields: - areaId - areaName - wasteBins (ArrayList)

Main behavior: - addWasteBin() - getWasteBins()

Waste (Abstract Class) Represents a general waste category. Field: -
wasteCategory

Subclasses: - OrganicWaste - PlasticWaste - MetalWaste - PaperWaste

Each subclass sets its own waste category using constructor chaining.

WasteBin Represents a garbage bin.

Fields: - binId - capacity - currentFillLevel - wasteType

Main behavior: - addWaste() - isFull() - emptyBin() -
getFillPercentage()

Truck Represents a garbage collection truck.

Fields: - truckId - capacity - currentLoad - assignedRoute

Main behavior: - loadWaste() - collectWasteFromBin() - assignRoute() -
isFull()

Route Represents a collection route for trucks.

Fields: - routeId - assignedTruck - areas (ArrayList)

Main behavior: - addArea() - assignTruck() - getAreas()

  ---------------------------------
  3. SERVICE LAYER IMPLEMENTATION
  ---------------------------------

The service layer contains manager classes that control system
operations.

RouteManager Responsible for managing routes.

Main behavior: - addRoute() - getRoutes() - findRouteById()

BinManager Responsible for monitoring waste bins.

Main behavior: - addBin() - getBins() - getFullBins() -
getBinsAboveThreshold()

FleetManager Responsible for managing trucks.

Main behavior: - addTruck() - getTrucks() - findTruckById()

ScheduleManager Responsible for managing collection schedules.

Uses: HashMap<String, ArrayList>

Meaning: day -> routes scheduled for that day

Main behavior: - addRouteToDay() - getRoutesForDay() - getScheduleMap()

  -----------------------------------
  4. SYSTEM SIMULATION (MAIN CLASS)
  -----------------------------------

The Main class simulates the operation of the waste collection system.

Simulation steps implemented:

1.  Create manager objects

    -   RouteManager
    -   BinManager
    -   FleetManager
    -   ScheduleManager

2.  Create Areas

3.  Create Waste Bins with different waste types

4.  Assign bins to areas

5.  Register bins in BinManager

6.  Create Trucks

7.  Register trucks in FleetManager

8.  Create Routes

9.  Add areas to routes

10. Assign trucks to routes

11. Register routes in RouteManager

12. Schedule routes for specific days

13. Simulate waste filling in bins

14. Detect bins above a collection threshold

15. Simulate truck collection of waste from bins

  ----------------------------
  5. CURRENT SYSTEM BEHAVIOR
  ----------------------------

The system currently simulates:

Citizens throw waste into bins ↓ Bins fill over time ↓ System monitors
fill levels ↓ Bins above threshold are detected ↓ Routes and trucks
collect waste ↓ Bins are emptied ↓ Truck load increases

This forms a simplified waste management simulation.
